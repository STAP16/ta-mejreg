Отчет: Модуль Б
Пакетная и потоковая обработка

1. Реализован алгоритм получения видеопотока
Для захвата видеопотока реализован модуль producer.py, который посредством библиотеки ultralytics получает кадры из видеофайла (резервный поток fallback2.mp4) в режиме stream=True. Обработка организована в цикле с построчным чтением кадров и немедленной передачей их в нейросетевую модель. Каждый кадр сопровождается генерацией метки времени (datetime.now().isoformat()) и сохранением кадра в объектное хранилище MinIO.

2. Выбран способ построения нейросетевой модели для распознавания объектов
В качестве детектора выбрана легковесная модель YOLOv8 Nano (yolo26n.pt), оптимальная для задач real-time детекции на CPU/GPU с ограниченными ресурсами. Модель настроена на распознавание 6 классов транспортных средств и пешеходов: pedestrian, bicycle, car, motorcycle, bus, truck (сопоставление индексов классов реализовано через словарь TARGET_CLASSES).

3. Выполнена оценка качества алгоритма путём расчёта метрики в соответствии со спецификой задачи
В процессе инференса накапливаются значения confidence (obj.conf) для каждого распознанного объекта. Каждые 30 кадров вычисляется и выводится средняя уверенность модели (avg_conf), что позволяет оперативно оценивать качество детекции на текущем видеопотоке.
Файл: metrics_validate.py расчитывает mAP50 модели в среднем от 0.7 и выше

4. Подключена и запущена модель детекции транспортных средств на потоковых данных
Модель запущена в режиме трекинга (model.track(..., stream=True, persist=True)). Результаты детекции в реальном времени сериализуются в JSON и публикуются в топик Kafka detections через confluent_kafka.Producer. Это обеспечивает потоковую передачу данных в downstream-системы.

5. Система распознаёт все объекты в кадре
Модель сконфигурирована на детекцию всех ключевых классов, присутствующих в дорожном потоке: легковые и грузовые автомобили, автобусы, мотоциклы, велосипеды и пешеходов. Каждый объект сопровождается уникальным идентификатором трека (track_id), полученным от модуля трекинга YOLO.

6. Система фиксирует ключевые параметры (тип средства, время, координаты)
Для каждого объекта формируется структурированное сообщение, содержащее:

camera_id — идентификатор камеры;
track_id — ID трека;
object_type — тип транспортного средства;
center_x, center_y — координаты центра ограничивающего прямоугольника в метрах (пересчёт из пикселей по коэффициенту PX_PER_METER = 40);
confidence — уверенность детекции;
timestamp — метка времени;
frame_path — ссылка на кадр в MinIO.
Все параметры сохраняются в таблицу bronze_detections ClickHouse.

7. В реальном времени рассчитываются и выводятся базовые метрики
В модуле producer.py реализован расчёт средней уверенности модели (avg_conf) с выводом в stdout каждые 30 кадров. Кроме того, конвейер потоковой загрузки (consumer.py) обеспечивает постоянное обновление сырого слоя данных (bronze) с частотой поступления сообщений из Kafka.

8. Реализован трекинг объектов между кадрами
Трекинг выполняется нативным механизмом ultralytics (persist=True). Уникальный track_id сохраняется в потоковых сообщениях и накапливается в ClickHouse, что позволяет на уровне silver-слоя отслеживать траекторию движения каждого объекта и рассчитывать его скорость по первой и последней точке трека.

9. Настроен и успешно запущен пакетный процесс пересчёта исторических данных
Оркестрация пакетных вычислений реализована в Apache Airflow. Создан DAG etl (dags/etl.py), который запускает задачу transform_bronze_to_silver. Реализована инкрементальная загрузка: скрипт запрашивает max(timestamp) из целевой таблицы silver_detections и подхватывает только новые данные из bronze_detections, обеспечивая идемпотентность и отказоустойчивость пересчёта исторических периодов.

10. Реализованы пакетные расчёты ключевых метрик с агрегацией по периодам
В модуле dags/build_mart.py реализовано построение витрины v_current_load с агрегацией по минутам. Рассчитываются следующие метрики:

Интенсивность — количество уникальных транспортных средств (uniqExact(track_id)), сгруппированных по минуте, камере и направлению;
Средняя скорость — avg(speed_kmh) по потоку;
Плотность потока — отношение числа ТС к условному участку дороги (uniqExact(track_id) / 0.5).
